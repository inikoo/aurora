Customizing and Exploring

When changing the styles in the template and customizing it for a project, a quick and useful tool to use is the Firebug plugin for Firefox. With it, you can inspect the elements on a webpage and see what CSS styling is being applied to them. Also, you can modify the CSS within Firebug to preview how changes to the styling will affect the site. Once you've tested in Firebug and know what changes need to be made, those same changes will need to be added to the stylsheet file. Firebug also shows where in the stylesheet the styles are located so you can more easily find the places in the stylsheet that need to be changed. A few resources for learning more about Firebug are the screecast videos at the Firebug website and this overview video about using Firebug for CSS.

Validation

The CSS validates as 2.1 except for the few CSS3 properties used to create rounded corners on elements and shadows beneath text.

Rounded Corners

The CSS3 border-radius property creates the rounded corners in all major browsers except Internet Explorer. IE 8 and earlier versions have no support for CSS3 border-radius so the corners are square in Internet Explorer. There are scripts that can give IE support for rounded corners but they have their limitations and drawbacks so I choose not to include one in the template. CSS3 Pie and CurvyCorners are two popular solutions.

Page with sidebar

To create a page with a sidebar, add an ID of “main-content” to the DIV containing the content that will be to the left of the sidebar. For the sidebar, a DIV with an ID of “sidebar” should be added directly below the “main-content” DIV. The DIV with a class of "clearfix" must be placed after the sidebar DIV or the layout will break.

IDs

main-content, sidebar

Full-width page without sidebar

To create a page without a sidebar where the main content fills the full width of the template, add an ID of “main-content-full” to the DIV that contains all of the main content on a page. You can reference the features.html file as an example of a full-width page. If you have elements at the bottom of the page content that are floated with CSS you'll need to add a DIV with a class of "clearfix" as in the code above for the sidebar.

main-content-full

Adding widgets to sidebar

To add content boxes to sidebar use a DIV with a class of “widget”. Within that DIV, there should be another DIV with a class of “widget-top” that has the text title for the widget, and a DIV with a class of “widget-content”. (The round-top class in the code rounds the corners of the top background in the widget)

Columns

Top

The template comes with many column options for organizing content: 1, 2, 3, 4, and 6 column. The number represents the total number of columns that will fit next to each other if there is no sidebar on a page and they can span the full 940px width of the layout. The columns can be used on pages with a sidebar as well, but not as many columns will fit next to each other in the smaller space.

For the 2,3,4, and 6 column layouts it is important that you add a class of “end-col” to the last DIV in a row so that the full number of columns will fit next to each other in the space. The column classes in the stylesheet each add margin to the right sides. The class “end-col” removes the right margin, so if it’s not added then the last DIV will overflow down to the next row below.

Classes

one-col, two-col, three-col, four-col, six-col

Dropdown Menu

Top

The main navigation is inside a DIV with an ID of “main-nav”. The class "sf-menu" doesn't do anything to style the menu, but it is necessary to make the dropdown animation work.To create a menu item, wrap an anchor tag that has a class of "nav-item" with a list tag. To display the item as active when on the page or a subpage of the menu, add an ID of "nav-selected" to the anchor tag. If you want the menu item to have a dropdown under it, the list tag must be given a class of "dropdown" and the unordered list within that list tag must have a class of "dropdown-wrap". To create submenu in the dropdown add a list tag with a class of "nav-flyout". To add another submenu, within the submenu, add a list tag with "nav-flyout" again (The CSS is setup so you can go a maximum of 3 sub-menus deep).

Classes

sf-menu, nav-item, dropdown, dropdown-wrap, nav-flyout

IDs

main-nav, nav-selected

Color Skins

Top

Each skin is created using a CSS file, “name of color skin.css”, that is separate from the main stylesheet. The images for each skin are in the images/skins folder. To change which skin is used, you’ll need to edit the line of code shown below at the top of each HTML file to point to the skin CSS file.

Custom Skins

Some of the skin design is controlled by the CSS in the “name of color skin.css” file. To change the images for the skin you can use one of the options listed below.

  • Open the skins_sliced.psd file included in the download. Make the changes and then go to the File menu in Photoshop and choose Save for Web. A dialog box will open up and from there you can choose “Save” and save the files in the images/skin folder you’re working on. The saved images resulting from the “Save” command will match the dimensions and file names of the images used in a skin. Next, go in to the CSS file for the skin and change the background-color and font-color values to go along with the skin images.
  • If there are different parts of skins that you like, you can combine the images and styles together from existing ones to make a new skin. Choose the existing skin that has the most elements that you like and start from there. For example, if you want to use the Sky Blue skin but change the logo area to be black from the Carbon skin, copy the file, “logo-bg.gif”, from the images/skins/carbon folder and paste the file into images/skins/blue.

Adding an image

Add a caption to slide

To add a caption that displays on top of one of the slides, add a Title attribute of “#htmlcaption” to the IMG tag. Then, create a DIV with an ID of “htmlcaption” and class of “nivo-html-caption” beneath the “nivo-slider” DIV and place the caption text into it. If you add more than one caption, make sure that the DIV ID of each one is unique. (You can name the ID for the DIV and the TITLE for the IMG whatever you want, but the two names must match and the TITLE must have a # mark before the name.)

prettyPhoto Lightbox

Top

This script displays a popup overlay window when an image is clicked on. It used on the features-detail, portfolio, and portfolio-3column pages. It can display just one image, or you can tell the script to group a number of images together as a gallery and allow the user to cycle through the images in the popup. The script also allows videos in the popup. Documentation on the lightbox, how to use it, and forum for questions or issues can be found at the prettyPhoto site.

Required
  • jquery-1.5.2.min.js
  • prettyPhoto.css
  • jquery.prettyPhoto.js
  • Javascript to start it on the page.
Settings

The settings for the lightbox can be found at the top of the HTML file. There, you can set the speed of the animation between images, the opacity of the color overlay window, and also change to different skins that come with prettyPhoto.

Setup

To use the lightbox with a single image, add an anchor tag that has a REL attribute of “lightbox”. The HREF is used to point to the content that you want to display in the lightbox. To add a title for the content above the lightbox that displays, use the ALT attribute. To add a caption, use the TITLE attribute.
Gallery

To group images into a gallery, use a REL attribute of “lightbox[name of gallery]” for each image you want in the gallery.

Contact Form

Top

The form is a PHP/AJAX mailer that will send the filled in form data to an email address. The PHP is a fallback so that the form will still work if the user has Javascript disabled in the browser. There’s a tutorial for this form in which the author explains how the form works and what the various files do. (Note: The name of the file ajax.js referenced in the tutorial was renamed to contact-form.js in the template files).

Required
  • jquery-1.5.2.min.js
  • contact-form.js
  • contact-send.php
  • email-validator.php
  • At least PHP 5 on your web hosting server. Most hosts these days are running PHP5. (The contact form in template was tested with PHP 5, but its possible it might work with an earlier version of PHP).

Setup

To set this form up, you just need to enter the email address to send the form to and also make sure that a current version of PHP is running on your web server. To enter an email address, go to line 45 of contact-send.php. If you're testing this locally on a computer without a server, the form will display an output error or nothing will happen when you try sending it because there will be no mail server. To setup and test the contact form is sending to the correct email address the site should be running on a hosting server.

If you try altering the form and run into issues or have questions, the best place to go for answers would be the tutorial for it.

Remove/Add Text in input fields

Top

This bit of Javascript removes the default value shown in a input field when a user clicks on the field. Its used on the search field form on the blog pages, so that when the field is clicked on, the text "Search Blog" is removed from the field. If the field is clicked on and nothing is entered into the field, then the text "Search Blog" is added back to the input when the user clicks away from the input field. To use this with an input field, there's a bit of javascript that needs to be placed in the head of the HTML file and a class of "hide-field-text" must be added to the input tag.

Required
  • Bit of javascript in the head of HTML file
  • Add class of "hide-field-text" to the input tag.

7. FAQs

Why are the corners not rounded in Internet Explorer?

IE8 and earlier versions do not support the CSS3 border-radius property. The corners will appear straight instead of the rounded corners you see in other browsers.

How do I make the corners rounded in IE?

There are a number of scripts out there that can create rounded corners in IE. The scripts usually do have some limitations as some elements in the template may require additional work for the script to work on them. CSS3 Pie and CurvyCorners are two popular solutions.

Where can I get the full icon sets used in the template?

  • Addictive Flavour Icons

  • spirit20 Icons

How do I change the logo and make the space for it bigger?

To increase the size of the colored area containing the logo, go to line 376 and 377 in style.css. Change width and height properties to the desired dimensions.

            #logo {
                width: 220px;
                height: 66px;
                display: block;
                float: left;
            }

To change the logo graphic, just replace the logo.png file with your own or if you want to change the file type to a JPEG or other format go to line 383 in style.css and make the change there.

            #logo a {
                background: url(../images/logo.png) no-repeat 18px 11px;
                height: 66px;
                display: block;
                text-indent: -9999px;
            }

What font was used to create the Elevate logo?

The font used was Museo Sans. The font can be purchased at MyFonts and a couple of the font weights are available for free.

Why is some Javascript functionality not working?

First, check that Javascript is not turned off in your browser. It's not very likely that it would be turned off, but its still good to rule it out. Then, look through section 3 in the documentation to check if there is any setup work that needs to be done for the functionality that’s not working.

If you changed the file structure by moving a folder around or renaming it and did not update the file path in the HTML that could also be the cause. Also, moving the order around in which the Javascript files are listed in the head of the HTML files would also cause some scripts to stop working. If either of those have been changed, try reverting them back to how things were originally setup in the downloaded files to see if that fixes the issue.

Updating to a version of jQuery that’s newer than what’s included with the template may also cause some issues. If that is the case, go to the sites listed in the Credits section and check for more recent releases of those scripts or try going back to the older version of jQuery to see if that fixes the issue.

Why are some of the images not showing in browsers?

First, check to see that all images needed have been uploaded to your server. Also, if any of the names of the folders or their locations relative to the other folders in the template files download from Themeforest have changed, check the HTML and CSS files to make sure the file paths to the changed image folder locations have been updated accordingly.

Why is the Twitter automatic linking and hovercard not working?

To make the script work properly when the files are uploaded to your server, you will need to follow these instructions and sign up for your own API key. If you have already done so, double check your domain settings by going back to Twitter and logging in. Also, check to make sure you added your API key to the line of code in the download files with the one given to you in the API signup process. If none that works, try going back to the original version of the download files if you've made any changes to them, add in your API key, and see if that will make it work.

It's also possible that the API service for it is down. If none of the above works, check out this Twitter API status page and see if any of them are down.
